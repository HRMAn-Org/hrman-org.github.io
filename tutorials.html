###[__Home__](index.html) : [__Sample data__](sample_data.html) : [__Downloads__](downloads.html) : [__Tutorials__](tutorials.html) : [__About us__](About_us.html) ###Installing KNIME and loading the HRMAn workflow <iframe width="640" height="360" src="https://www.youtube.com/embed/bCf17RR0yBA" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe> KNIME is an open source data analytics, reporting and integration platform. KNIME integrates various components for machine learning and data mining through a modular data pipelining concept. A graphical user interface allows assembly of nodes for data preprocessing, for modeling and data analysis and visualization without, or with only minimal, programming. KNIME can be downloaded [__here__](https://www.knime.com/downloads). If you want to learn more about KNIME, you can check their [__homepage__](https://www.knime.com/). It also includes a [__forum__](https://www.knime.com/forum) which can be useful if you should encouter any issues with it. The video above shows you how to install KNIME and extensions needed to run HRMAn on your computer. After you have completed the steps you should be able to execute HRMAn and analyse your imaging data. The next video will provide you with a hands-on demo on how to use HRMAn. ###Using HRMAn <iframe width="640" height="360" src="https://www.youtube.com/embed/JnwYZsx4kn4" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe> This video will guide you through your first image analysis with HRMAn. Please refer to the other videos to find more information on how to set up KNIME and how to prepare the training datasets. If you want to run the analysis on your computer you can download the example data [here](https://bitbucket.org/hrman/hrman-src/raw/67c1e521090492ecfe447dfda4d9f550c62ccd48/Tutorial_data.zip). ###Pre-processing of microscope images <iframe width="640" height="360" src="https://www.youtube.com/embed/HBuAlTruYGk" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe> The video will explain to you how to pre-process your images if they were not imaged on a high content imaging microscope. You can use any microscope that is available to you and use this KNIME pipeline to re-name your images so that they can be clustered appropriately. The KNIME workflow can be downloaded [here](downloads.html). ###KNIME deep learning for HRMAn <iframe width="640" height="360" src="https://www.youtube.com/embed/Ak1JIYIcbIg" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe> This video will show you how to set up DL4J in KNIME and how to use it to train the AlexNet deep convolution network. Once you have trained your network foryour pathogen and staining, you can save it and use it in the HRMAn pipeline to analyse your images. In order to use GPU training in KNIME for your model you need to have a Nvidia graphics card and CUDA installed. CUDA can be downloaded [here](https://developer.nvidia.com/cuda-80-ga2-download-archive) and a good explanation how to install it can be found [here](https://www.quantstart.com/articles/Installing-Nvidia-CUDA-on-Mac-OSX-for-GPU-Based-Parallel-Computing) (For Mac). ###Creating your own learning dataset for machine-learning based prediction of pathogens per vacuole <iframe width="640" height="360" src="https://www.youtube.com/embed/7Vt0JaAwTXU" frameborder="0" allow="autoplay; encrypted-media" allowfullscreen></iframe> In order to run HRMAn with your prefered pathogen and to do prediction of pathogens per vacuole/cluster you will need to create a reliable dataset for the machine learning algorithm integrated into HRMAn. This workflow will crop the images for you and prepare the spreadsheet file, so that only one column containing the value to predict has to be filled in based on the cropped images. The video will show you how to do this. The workflow can be downloaded [here](downloads.html).
